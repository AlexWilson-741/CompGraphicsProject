# Computational Graphics Project

Brief Overview:
In this project, I designed a 3D scene from the ground up, using various meshes, textures, shaders, and lighting.  I also implemented keyboard and mouse camera movement with adjustable sensitivity and movement speed.
---
My approach to designing software in this project was to start simple and then refine based on what broke visually. I didn’t try to design everything perfectly upfront, rather I built a basic 3D scene and adjusted it as issues became obvious. One of the main skills I developed was understanding that geometry isn’t independent. Scaling or moving one part of an object affects everything around it. For example, resizing objects like the chair required adjusting positions and proportions together to avoid distortion and clipping. Because of that, my design process became more about maintaining relationships between objects rather than just placing them individually. This type of problem solving can carry over into future work by focusing on how components interact within a system instead of treating them as isolated pieces.

From a development standpoint, I moved away from trial and error and toward reasoning about transformations so the scene behaved as expected. Iteration was initially a major part of the process. I'd build something, observe what looked wrong, and then refine it in a more controlled way. Over time, I became more intentional with how I structured objects and applied transformations, which made the scene easier to adjust and prevented repeated issues. This project also strengthened my ability to think in terms of space and geometry, which is useful both academically and professionally. In future coursework, these skills will likely apply directly to areas like graphics, simulation, and visualization. Professionally, they translate to any field involving 3D environments or perhaps game development, where understanding spatial relationships is quite important.
